## INTERVIEW QUESTIONS BY CANDIDATE TYPE

### For HUNTER (Persistence Hunter)
1. "What registry persistence technique do you consider most dangerous and why?"
2. "Design a detection system for IFEO tampering that generates <1% false positives."
3. "How would you hunt for fileless malware that only uses registry for execution?"
4. "What's your favorite registry-based evasion technique and how would you detect it?"

### For ARCHITECT (Infrastructure Lead)
1. "Design a registry configuration system for 50,000 global endpoints."
2. "How would you implement transaction-based registry operations spanning multiple hives?"
3. "A critical registry corruption has occurred on a production domain controller. Walk me through recovery."
4. "Design a registry backup system with <5 second RTO."

### For SAFETY (QA Engineer)
1. "Design a test strategy for a registry modification that must work across Windows 10-11."
2. "How would you safely test a registry change that affects system boot?"
3. "What's your process for creating a registry sandbox that can be reverted instantly?"
4. "A test causes blue screens on 30% of machines. What's your diagnosis process?"

### For AUTOMATOR (DevOps Engineer)
1. "Walk me through safely deploying a registry change across 5,000 workstations."
2. "Design a registry versioning system with automatic rollback capability."
3. "How would you implement registry compliance reporting across 10,000 endpoints?"
4. "A critical registry change is causing 20% application crash rate. What's your response?"

### For PREDICTOR (AI/ML Specialist)
1. "Design an ML pipeline to detect malicious registry modifications in real-time."
2. "How would you baseline 'normal' registry activity across 10,000 endpoints?"

4. "How would you optimize a registry anomaly detection model for <50ms latency?"
